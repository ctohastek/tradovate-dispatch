Project File Structure

Organized layout for the dispatcher app

tradovate-dispatch/
├── app/
│ ├── main.py                          # FastAPI app entry point
│ ├── config.py                          # Config loading (env vars, etc.)
│ ├── database.py                        # SQLite connection & init
│ ├── models.py                          # Pydantic models for request/response
│ ├── auth/
│ │ └── api_key.py                    # Agent key validation
│ ├── parser/
│ │ ├── grammar.lark                  # Command grammar
│ │ ├── parser.py                      # lark parser + parse logic
│ │ └── validator.py                  # Semantic validation (qty, price)
│ ├── tradovate/
│ │ ├── client.py                      # HTTP client + auth token management
│ │ └── commands.py                  # Maps parsed command → Tradovate API call
│ ├── rate_limit/
│ │ └── limiter.py                    # Per-agent rate limit tracking
│ ├── alerts/
│ │ └── mailer.py                      # Email alerts
│ ├── logging/
│ │ └── audit.py                       # Audit logging to DB + file
│ └── routes/
│ ├── execute.py                      # POST /execute endpoint
│ ├── health.py                       # GET /health endpoint
│ └── admin.py                         # (future) Admin endpoints
├── tests/
│ ├── test_parser.py                      # Grammar + parsing tests
│ ├── test_validator.py                  # Validation tests
│ ├── test_rate_limit.py                # Rate limit logic tests
│ ├── test_auth.py                        # API key validation tests
│ └── test_integration.py               # Full flow with mock Tradovate
├── docs/
│ ├── SETUP.md                            # Installation & setup guide
│ ├── COMMAND_REFERENCE.md              # All supported commands + examples
│ ├── CURL_EXAMPLES.md                  # curl command templates
│ ├── API.md                              # API endpoints & responses
│ ├── DEPLOYMENT.md                        # Deployment checklist
│ └── TROUBLESHOOTING.md                # Common issues & fixes
├── config/
│ ├── agents.example.yaml                # Example agent config
│ └── dispatcher.example.env            # Example env vars
├── logs/
│ └── (rotating log files created at runtime)
├── requirements.txt                            # Python dependencies
├── dispatcher.db                                  # SQLite database (created on init)
├── gunicorn.conf.py                                # Gunicorn config
├── run.py                                            # Entry point (loads config, starts app)
└── README.md                                      # Project overview

Key principles:

  • Modular: Each responsibility in its own module (auth, parser, rate limiting, etc.)
  • Testable: No hard dependencies; each component can be tested in isolation
  • Documented: Comprehensive docs for agent setup, deployment, troubleshooting
  • Configurable: Agent configs in YAML; env vars for dispatcher settings
