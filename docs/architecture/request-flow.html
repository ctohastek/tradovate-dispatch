Request/Response Flow

How a command travels through the dispatcher

Agent sends: POST /execute
curl -X POST http://localhost:8000/execute \
  -H "Authorization: Bearer AGENT_API_KEY" \
  -d 'BUY 2 ES LIMIT 6425.50 SET STOP 6400.00 SET TAKE PROFIT 6500.00'
↓
Step 1: Authenticate

✓ API Key valid? Look up agent in DB

✓ Agent exists? Retrieve: target_url, rate_limit, Tradovate creds

✗ If invalid: Return 401 + log

↓
Step 2: Parse Command

Grammar check: BUY 2 ES LIMIT 6425.50 ...

Semantic validation: qty in [1-999]? prices in XXXX.XX format?

Output:

{
  "command_type": "BUY",
  "quantity": 2,
  "contract": "ES",
  "limit_price": 6425.50,
  "stop_loss": 6400.00,
  "take_profit": 6500.00
}

✗ If parse fails: Return 400 + clear error message

↓
Step 3: Check Rate Limit

Count requests: agent's requests in last 60 seconds

Check against limit: rate_limit_per_minute

✓ Within limit? Proceed to Step 4

✗ Rate limited? Return 429 "WAITING" + send alert email to admin

↓
Step 4: Tradovate Auth (if needed)

Check cached token: Is token valid and < 75 min old?

✓ Yes: Use cached token

✗ No: Call /auth/accessTokenRequest with agent's Tradovate creds

Store token: Cache it for next request

↓
Step 5: Call Tradovate API

Construct request: Map parsed command → Tradovate order format

Send to: agent's target_url (locked, cannot change)

Example: POST /order/placeorder

{"orderType": "LMT", "quantity": 2, "price": 6425.50, ...}

↓
Step 6: Process Response

✓ Success (200): Extract orderId, log to audit_logs, return JSON to agent

✗ Failure (4xx/5xx): Log error details, send alert email to admin, return error to agent

↓
Agent receives response
{
  "success": true,
  "orderId": "12345",
  "message": "Order placed: 2 ES @ LIMIT 6425.50",
  "tradovateResponse": {...}
}
